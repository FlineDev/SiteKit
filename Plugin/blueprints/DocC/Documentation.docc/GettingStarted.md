# Getting Started

Set up the project and build your first documentation page in a couple of minutes.

@Metadata {
   @TitleHeading("Guide")
   @PageKind(article)
}

> **Quick Read**:
> _Install the toolchain, run the dev server, and edit a Markdown note – the page rebuilds as static, AI-fetchable HTML._

## Install

Add your notes as Markdown files inside `Documentation.docc/`. Each file is one page; DocC directives like `@Metadata`, `@Image`, and `@Row`/`@Column` are supported alongside standard Markdown.

## Cross-references

Link between notes with a DocC reference – for example, see <doc:Reference> for the full option list. SiteKit resolves it to the target page's URL automatically.
