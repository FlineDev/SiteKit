# Italian (it) – Web Content Style Guide

Read `principles.md` first. Italian-specific rules for SiteKit content.

## Address & formality
- **"tu"** suits personal, indie, and casual product sites. **"Lei"** (capitalized in the courtesy sense) is formal/business. Decide once and keep it. Italian often phrases impersonally (`è possibile…`) to avoid choosing.

## Quotation marks
- Use caporali « … » for primary quotes, or “ … ” – be consistent. Nested quotes switch style.

## Punctuation & spacing
- No space before `: ; ! ?`. Accents matter and are not optional: `è` (is) vs `e` (and), `perché`, `città`, `così` – use the correct grave/acute accent.

## Numbers & dates
- Decimal **comma**, thousands **point**: `1.234,50 €` (symbol after, with space). Don't reformat numbers inside placeholders.
- Dates: `14 giugno 2026` – **months lowercase**, day first.

## Capitalization & terminology
- **Sentence case** for headings; do not capitalize months, weekdays, or languages (`italiano`, `lunedì`, `giugno`).
- Common web terms: `articolo`/`post` (post), `iscriviti` (subscribe), kept `newsletter`, `privacy` (kept), `note legali` (legal notice).

## Common pitfalls
- False friends: "attuale" = *current* (not *actual* → `reale/effettivo`); "eventualmente" = *possibly* (not *eventually*); "libreria" = *bookshop* (a code library is `libreria` too but context differs).
- Apostrophes and elisions: `l'articolo`, `un'app` (feminine), `dell'utente`.
