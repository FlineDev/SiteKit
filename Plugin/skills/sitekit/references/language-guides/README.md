## A language not listed here?

Use [`principles.md`](principles.md) (it covers the categories that matter for any language) plus, for an Apple-platform audience, Apple's per-language localization style guides shipped with Xcode 26+ – they cover **ar, de, fi, fr, fr-CA, he, hi, it, ja, ms, nb, sv, uk, zh-Hans** at app-string depth. To add a dedicated guide, copy the structure of an existing file (address & formality, quotation marks, punctuation & spacing, numbers & dates, capitalization & terminology, common pitfalls) and fill in the target language's conventions for web prose.

## On sources

These guides were written for SiteKit from each language's web-writing conventions. Apple's Xcode 26+ localization style guides were consulted to confirm which conventions matter; they remain the authoritative reference for app-string-level detail and are not reproduced here.
